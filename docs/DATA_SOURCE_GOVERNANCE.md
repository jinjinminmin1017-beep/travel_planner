# 数据源治理当前说明

本文件取代早期数据源治理草案。当前治理原则如下：

1. 数据源必须在 `DataSourceConfig` 中登记，并通过 `/api/data-sources/status` 暴露运行状态。
2. 真实 Provider 必须满足启用开关、授权状态和必要环境变量后才可参与业务路径。
3. 缺少 key、授权未完成、调用失败或返回空结果时，必须返回降级状态、业务错误、`SourceFailure` 或阻断对应方案类型。
4. 运行时不得把测试交通事实当作真实结果返回。
5. `DataSourceConfig` 不得通过状态接口暴露密钥、QPS 细节或商业配置。
6. 生产环境只能启用授权通过的数据源。
7. 任何 Redirect-only 能力都不得携带登录、下单、支付、抢票参数。
8. LLM 只能做文本解析、推荐选择和解释，不得生成路线、价格、车次、航班、余票或购买链接。
